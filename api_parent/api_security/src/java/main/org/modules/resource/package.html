This is the module that provides classes and interfaces for accessing and processing data pertaining to the application resources, their types, and sub types. In the context of the security API, a resource basically exists as a component or service that is made available to any API. At runtime, resources are generally housed in an in memory collection call the Resource Registry. This registry is the data source for any API to extract information regarding a resource propreties.
A resource can be subclassed into resource types. An example of resouce types would be:
  • web services
  • forms
  • form components
  • servlets
  • menus
  • menu items

Moreover, resource types can be subclassed into resource sub types. An example of sub types for web service resources would be:
  • HTML
  • RMI
  • SOAP
  • XML
  • Sockets
  • COM
